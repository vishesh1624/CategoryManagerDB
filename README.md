# Fruits & Vegetables Category Command Center

A web-based command center that simulates how a quick commerce or e-commerce category manager can track, diagnose, and improve performance for a fruits and vegetables category.

## Live Demo
https://category-manager-db.vercel.app/

## What this project helps users understand

This project is designed to show how category performance can be monitored across key business, supply, and customer metrics.

It helps users explore questions such as:

* Which products or sub-categories need attention?
* Where is demand strong but availability weak?
* Which SKUs may be causing margin pressure?
* How are freshness, stock availability, and revenue connected?
* What actions can a category manager take based on performance signals?

## Who this is useful for

This project is useful for:

* Recruiters evaluating product, category, or growth-oriented portfolios
* Category managers in quick commerce or e-commerce
* Product managers working on internal business dashboards
* Business analysts learning category performance tracking
* Students or professionals showcasing commercial problem-solving skills

## Key features

The command center presents a practical view of category management through:

* Category-level performance overview
* SKU and sub-category level tracking
* Revenue and margin indicators
* Stock availability and inventory signals
* Freshness and quality-related metrics
* Action-oriented insights for category decisions

## How to use the command center

### 1. Review the category overview

Start with the main dashboard to understand overall category performance across revenue, availability, freshness, and margin indicators.

### 2. Identify problem areas

Look for SKUs or sub-categories where performance is weak. For example, a product may have high demand but poor availability, or strong sales but low margin.

### 3. Read the business signals

Use the dashboard signals to understand what may be driving performance. These could include stock-outs, freshness issues, weak demand, or margin pressure.

### 4. Translate insights into actions

The project is structured to show how a category manager can move from data to decisions, such as improving replenishment, prioritizing high-demand SKUs, reviewing pricing, or addressing quality issues.

## Example use cases

### Monitoring daily category health

A category manager can use the dashboard-style interface to quickly review whether the fruits and vegetables category is performing well across commercial and operational metrics.

### Diagnosing SKU-level issues

The command center can help identify products that may require intervention due to low availability, poor freshness, weak sales, or margin concerns.

### Supporting business reviews

The project can be used as a portfolio example to demonstrate how category data can be converted into clear, decision-ready insights.

### Demonstrating quick commerce thinking

The simulator reflects the kind of trade-offs common in quick commerce, where availability, freshness, pricing, and demand need to be managed together.

## Important notes

This is a static portfolio project created for demonstration and learning purposes. The data and logic are simulated and should not be treated as live business data.

The project is intended to demonstrate category management thinking, dashboard design, and business decision-making for quick commerce or e-commerce use cases.

## Privacy

This is a front-end-only static web app. It does not require login and does not store user-entered data on a server.
